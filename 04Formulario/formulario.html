Todos los formularios en la web, manejan métodos para el envío de las diferentes peticiones que realiza el usuario considerando la arquitectura cliente servidor, el manejo de las peticiones se hace por medio del protocolo http y actualmente pot https, cada petición debe ser atendida en un tiempo de respuesta conforme al modelo TCP/IP. Los métodos principales de los formularios son:

  1. GET:

    Este método se encarga de enviar la información del formulario por medio de la salida estandar (conocida como url), del navegador, esto propicia que puedan existir inyecciones de código realizando violaciones a la seguridad del sistema.

  2. POST:

    Este método se encarga de enviar la información para poder registrar un cambio o un nuevo registro dentro del sistema, enviandola por el metodo estandar; pero mediante un ocultamiento, esto significa que no es visible para el usuario. Lo que nos ofrece un mayor grado de seguridad en el manejo de la información.


En resumen los métodos son:

  • GET: Sirve para enviar y realizar operaciones de lectura o consulta
  • POST: Sirve para enviar información de un recurso al servidor, con el fin de registrarlo
  • Delete: Sirve para eliminar un recurso específico dentro del servidor, obteniendo una respuesta de actualización
  • PUT: Sirve para alamcenar la información de un recurso particular y conectarse al servicio para realizar la actualización
  • HEAD: Sirve para regresar el codigo de puesta a cualquier cabecera HTTP asociado a un recurso
